# Research: FinOps - 2026-03-03

**Time:** 1800 UTC  
**Topic:** Snowflake FinOps Cost Optimization  
**Researcher:** Snow (AI Assistant)

## Accurate Takeaways

1. Snowflake added an `ACCOUNT_USAGE` view, `CORTEX_AI_FUNCTIONS_USAGE_HISTORY`, to monitor Cortex AI Functions credit consumption with dimensions including function, model, user, role, warehouse, and query. (GA as of Mar 2, 2026.)
2. The AI Functions cost telemetry is explicitly positioned to support automated controls like monthly spend thresholds, per-user limits (revoke/restore access), and identifying/canceling runaway AI-function queries.
3. Snowflake simplified hybrid table pricing: as of Mar 1, 2026 hybrid table “request” charges (serverless credits for reads/writes on underlying row storage) are removed; hybrid tables now bill only storage (flat monthly per-GB) + standard warehouse compute.
4. Snowflake expanded “owner’s rights contexts” (owner’s rights stored procedures, Native Apps, Streamlit) to allow more introspection: most `SHOW`/`DESCRIBE` plus `INFORMATION_SCHEMA` views/table functions are now accessible; exceptions include query/login history functions.
5. Snowflake Native Apps introduced (Preview) two provider-to-consumer interaction primitives that reduce ad-hoc setup patterns: (a) “application configurations” for requesting consumer-provided values (including sensitive values), and (b) inter-app communication between apps within the same consumer account.

3. **Native App introspection in owner’s-rights:** enable in-app diagnostic pages that run `SHOW`/`DESCRIBE`/`INFORMATION_SCHEMA` queries safely (e.g., schema discovery, object existence checks) without requiring extra consumer grants—while keeping history data as a privileged/optional path.

## Concrete Artifacts

### 1) Draft: monthly AI Functions spend by user/role

*(Column names may vary—verify against the view reference once you wire it up.)*

```sql
-- Monthly Cortex AI Functions credits by user + role
SELECT
  DATE_TRUNC('month', start_time) AS month,
  user_name,
  role_name,
  SUM(credits) AS credits
FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_FUNCTIONS_USAGE_HISTORY
WHERE start_time >= DATE_TRUNC('month', CURRENT_DATE())
GROUP BY 1,2,3
ORDER BY credits DESC;
```

### 2) Draft: detect runaway AI Functions queries (top N by credits)

```sql
SELECT
  query_id,
  user_name,
  role_name,
  warehouse_name,
  model_name,
  function_name,
  credits,
  start_time
FROM SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_FUNCTIONS_USAGE_HISTORY
WHERE start_time >= DATEADD('day', -1, CURRENT_TIMESTAMP())
ORDER BY credits DESC
LIMIT 50;
```
